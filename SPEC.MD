# PatchGlow — Browser-Based Modular Synth Emulator  ## Full Project Specification for Claude Code  -----  ## 1. Project Vision  PatchGlow is an open-source, browser-based modular Eurorack synthesizer emulator built as an **interactive visual learning tool for synthesis**. Every module visualizes what it does in real time. Every patch cable shows what signal is flowing through it. The aesthetic is retro-futurist — dark panels, neon glow, CRT warmth — like patching a synthesizer on the bridge of a 1982 sci-fi starship.  The target user knows little about synthesis and learns by patching, turning knobs, and **watching what happens**. The guiding design principle: **if a knob turns and nothing visibly changes, either the knob shouldn't exist or the visualization is missing something.**  ### Core Design Tenets  1. **Visualization IS the feature** — every module has a real-time display showing exactly what it does to signal 1. **Sound is immediate** — patch two things together, hear the result, no menus or config 1. **The aesthetic earns attention** — retro-futurist neon glow, not clinical developer-tool UI 1. **Modules teach concepts** — each module exists to teach ONE synthesis concept clearly 1. **Patch cables tell the story** — color-coded, animated signal flow shows audio vs CV vs gate  ### License  MIT — compatible with Tone.js, Mutable Instruments source, and all dependencies.  -----  ## 2. Each cable is a cubic Bézier curve with a natural droop (gravity sag).  **Cable Path Calculation:**  ```typescript function getCablePath(   start: { x: number; y: number },   end: { x: number; y: number } ): string {   const dx = end.x - start.x;   const dy = end.y - start.y;   const dist = Math.sqrt(dx * dx + dy * dy);    // Droop increases with horizontal distance, subtle for short cables   const droop = Math.min(dist * 0.3, 120);    // Control points create a hanging curve   const cp1x = start.x + dx * 0.25;   const cp1y = start.y + droop;   const cp2x = start.x + dx * 0.75;   const cp2y = end.y + droop;    return `M ${start.x} ${start.y} C ${cp1x} ${cp1y}, ${cp2x} ${cp2y}, ${end.x} ${end.y}`; } ```  **Cable Colors** by signal type:  - Audio: `--color-audio` (#ff6b35, warm amber) - CV: `--color-cv` (#00e5ff, cyan) - Gate: `--color-gate` (#ff2ecb, magenta)  **Cable Rendering**: Each cable is a `<path>` with:  - A thicker (4px) semi-transparent background stroke for body - A thinner (2px) brighter stroke on top for the "core" - A subtle `filter: drop-shadow()` glow in the cable color  **Signal Flow Animation**: Animated dashes or dots traveling along the cable path, from output to input. Use SVG `stroke-dasharray` + `stroke-dashoffset` animation:  ```css .cable-flow {   stroke-dasharray: 8 16;   animation: flow 1s linear infinite; } @keyframes flow {   to { stroke-dashoffset: -24; } } ```  The animation speed should vary by signal type:  - **Audio**: Fast flowing particles (represents high-frequency signal) - **CV**: Slow, smooth undulation (represents slow modulation) - **Gate**: Discrete pulses/flashes (represents on/off triggers)  For audio cables, consider pulsing the cable's brightness/opacity in sync with the signal amplitude — louder signal = brighter cable. This can be driven by an analyser node's RMS value.  **Cable Interaction:**  - Drag from any output port → cable follows cursor → release on valid input port to connect - While dragging, the in-progress cable is rendered with a dashed style - Valid destination ports pulse/glow to indicate compatibility - Click on a cable to select it (highlight brighter), press Delete/Backspace to remove - Right-click on a cable for a context menu with "Remove" option  ### 4.6 Rack Grid  The rack is a horizontal grid measured in **HP (Horizontal Pitch)** units. Standard Eurorack: 1HP = 5.08mm. For the browser, scale to pixels: **1HP = ~20px** (adjust for screen size).  The rack has **rails** at top and bottom — thin horizontal bars in `--rack-rail` color with regularly spaced screw holes. The background between rails is `--bg-rack`.  Standard rack height is **3U** (three "units" of 5.25" each). For simplicity, start with a **single row** rack. Modules snap to HP-aligned positions when placed.  Empty rack space should have a subtle visual — faint grid lines at every HP, or a very dim "place module here" pattern.  ### 4.7 CRT / Atmosphere Effects  Apply globally (or as a toggle in settings):  - **Vignette**: Subtle darkening at screen edges via a CSS `radial-gradient` overlay - **Scanlines**: Very faint horizontal lines (see §4.4 CSS above) - **Subtle glow bloom**: On the body/html, a very faint `backdrop-filter: blur(0.5px)` or `filter` to soften bright elements slightly, creating a phosphor-glow feel - **Ambient particles** (optional, Phase 5+): Very slow, tiny floating specks in the background — like dust in a dark studio  -----  ## 5. Module Specifications  ### 5.1 Oscillator — "What is sound?"  **Concept taught**: Sound is a repeating waveform. Different shapes have different harmonic content. When user switches from sine (one peak) to sawtooth (many harmonics), the spectral change is immediately visible and dramatic. - Waveform trace should have a subtle phosphor-glow trail effect (draw with reduced opacity over previous frames, or use `globalAlpha` fade).  ### 5.2 Filter — "What is filtering?"  **Concept taught**: A filter removes frequencies from a sound. Cutoff determines where. Resonance emphasizes frequencies at the cutoff point.  |Property|Value                    | |--------|-------------------------| |HP Width|12                       | |Category|modifier                 | |Accent  |`--accent-filter` (amber)|  **Parameters:**  |ID       |Label |Type  |Range                      |Default|Curve      |Unit| |---------|------|------|---------------------------|-------|-----------|----| |frequency|CUTOFF|knob  |20–20000                   |1000   |exponential|Hz  | |resonance|RES   |knob  |0–30                       |1      |linear     |—   | |type     |TYPE  |select|lowpass, highpass, bandpass|lowpass|—          |—   |  **Ports:**  |ID       |Label  |Direction|Signal Type| |---------|-------|---------|-----------| |in       |IN     |input    |audio      | |cutoff-cv|FREQ CV|input    |cv         | |out      |OUT    |output   |audio      |  **Audio Engine:**  - `Tone.Filter` with type, frequency, Q (resonance) parameters - CV input modulates the frequency parameter  **Visualization:**  - **Frequency response curve**: The classic EQ-style curve. Draw the filter's theoretical response curve based on current type, cutoff, and resonance. - As the user sweeps cutoff, the curve slides left/right. Interaction Design Details  ### 6.1 Knob Interaction  - **Pointer lock**: On mousedown on a knob, capture the pointer. Vertical mouse movement (up = increase, down = decrease) controls the value. This prevents the cursor from flying off the knob. - **Sensitivity**: Normal drag = full range over ~300px of movement. Hold Shift for fine control (10x slower). - **Visual feedback**: While turning, the knob's glow ring brightens. The value readout updates in real time. - **Double-click**: Reset to default value. - **Scroll wheel**: Also adjusts knob value when hovering over it.  ### 6.2 Cable Interaction  - **Creating**: Click and drag from an output port. A preview cable follows the cursor. Valid destination ports (matching compatibility rules) glow/pulse. Release on a valid port to complete the connection. - **Removing**: Click a cable to select it (cable brightens, slight "lift" animation). Press Delete key or click a "remove" button that appears. Or: right-click → "Remove cable." - **Compatibility**: Audio outputs can connect to audio inputs. CV outputs can connect to CV inputs. Gate outputs connect to gate inputs. Cross-type connections could be allowed with a visual warning (dimmer cable) since in real Eurorack, signals are interchangeable — this is a teaching moment about voltage. - **Multiple cables**: Multiple cables can connect to the same port (summing behavior on inputs, fan-out on outputs), matching real Eurorack behavior.  ### 6.3 Module Placement  - **From library**: Drag a module card from the sidebar onto the rack. Ghost/preview shows where it will land. Snaps to HP grid. If there isn't enough space, show a red indicator. - **Repositioning**: Drag an existing module to move it within the rack. Other modules do NOT auto-reflow (like real Eurorack — you must manually arrange). - **Removing**: Right-click module → "Remove from rack." All connected cables are removed too. Audio engine disposes.  ### 6.4 Global Controls (Toolbar)  - **Start Audio** button: Required due to browser autoplay policy. Big, obvious, glowing. `Tone.start()`. - **Master Volume**: Global output knob/slider. - **Patch Management**: Save (to localStorage), Load, Export (download JSON), Import (upload JSON). - **Module Library Toggle**: Show/hide the sidebar.  -----  ## 7. Performance Considerations  - **Canvas rendering**: Each module's visualization should use its own `<canvas>` element and render via `requestAnimationFrame`. Use a shared RAF loop to batch all canvas draws per frame, not one RAF per module. - **Analyser nodes**: Tone.Analyser nodes are cheap but add up. Share analysers where possible (e.g., one per output port, not one per module). - **Audio thread**: Never do DOM manipulation in audio callbacks. All Tone.js scheduling stays on the audio thread; visualization reads from analysers on the main thread. - **Cable rendering**: SVG with a reasonable number of cables (< 30) is fine. If performance becomes an issue, switch cables to Canvas rendering. - **Module limit**: Practically, 15–20 modules should work smoothly on a modern machine. Consider showing a friendly warning if the user exceeds this.  -----  ## 9.
