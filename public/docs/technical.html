01 Architecture Overview

PatchGlow follows a unidirectional data flow with a clean separation between state, UI, and audio processing. The Zustand store is the single source of truth.


┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  Zustand     │────▶│  React UI    │────▶│  User sees   │
│  Store       │     │  Components  │     │  & interacts │
│              │◀────│              │◀────│              │
└──────┬───────┘     └──────────────┘     └─────────────┘
       │
       │ subscribes
       ▼
┌─────────────┐
│  Audio       │
│  Engine      │
│  (Tone.js)   │
└─────────────┘

Key Principles

  • The Zustand store (src/store/rackStore.ts) is the single source of truth for all rack state: modules, connections, parameters, and UI state.
  • The AudioEngineManager (src/engine/AudioEngineManager.ts) is a singleton that subscribes to store changes and keeps the Tone.js audio graph in sync.
  • React components render from store state and dispatch actions back to it. No component holds audio state.
  • Visualization data flows from Tone.js Analyser nodes through AudioEngineManager.getVisualizationData() into Canvas rendering at 60fps.

04 Module System

The module system is the core extensibility mechanism. Every module implements three parts, packaged together as a ModuleRegistration.

ModuleDefinition

Defined in src/types/module.ts. Describes the module's interface — ports, parameters, and metadata.

interface ModuleDefinition {
  type: string           // Unique identifier, e.g. 'oscillator'
  name: string           // Display name
  description: string    // Short description
  learningSummary: string // What synthesis concept this teaches
  hp: number             // Width in HP (1HP = 20px)
  category: 'source' | 'modifier' | 'modulation' | 'utility' | 'output'
  ports: PortDefinition[]
  parameters: ParameterDefinition[]
  accentColor: string    // Hex color for panel accent
}

PortDefinition

{ id, label, direction: 'input' | 'output', signalType: 'audio' | 'cv' | 'gate', description? }

ParameterDefinition

{ id, label, type: 'knob' | 'switch' | 'select',
  min?, max?, default, step?, unit?,
  curve?: 'linear' | 'exponential' | 'logarithmic',
  options? }

ModuleAudioEngine

Defined in src/types/module.ts. Manages the Tone.js audio nodes for a single module instance.

interface ModuleAudioEngine {
  initialize(context: Tone.BaseContext): void
  getOutputNode(portId: string): Tone.ToneAudioNode | Tone.Signal
  getInputNode(portId: string): Tone.ToneAudioNode | Tone.Param | Tone.Signal
  setParameter(parameterId: string, value: number | string): void
  getVisualizationData(): VisualizationData
  dispose(): void
  handleAction?(action: string, payload?: unknown): void
  onPortConnected?(portId: string): void
  onPortDisconnected?(portId: string): void
}

VisualizationComponent

A React component receiving { moduleId, data: VisualizationData, accentColor }. Renders onto a <canvas> element using the shared useAnimationFrame hook from src/modules/_shared/useAnimationFrame.ts.

ModuleRegistration

Bundles all three parts into a single export.

{ definition, createEngine: () => ModuleAudioEngine, VisualizationComponent }

05 Audio Engine

AudioEngineManager (src/engine/AudioEngineManager.ts) is a singleton that bridges the Zustand store and Tone.js.

init()

Syncs initial state, then subscribes to three store selectors:

  1. state.modules → syncModules() — create/dispose engines
  2. state.connections → syncConnections() — connect/disconnect Tone.js nodes
  3. state.modules (again) → detect parameter changes by deep comparison

syncModules()

Iterates store modules. Creates engines for new modules via registration.createEngine() → engine.initialize() → apply default params. Disposes engines for removed modules.

syncConnections()

Diffs old vs new connection arrays:

  • Removed connections: sourceEngine.getOutputNode().disconnect(destEngine.getInputNode())
  • New connections: outputNode.connect(inputNode)
  • Notifies engines via onPortConnected / onPortDisconnected so modules can adjust internal state (e.g., VCA reconnects internal bias when external CV is disconnected).

Parameter Propagation

Store change → subscribe callback compares param values → engine.setParameter(id, value) → engine sets Tone.js AudioParam.

Visualization

getVisualizationData(instanceId) returns the engine's current analyser/state data. Called at ~60fps by visualization components via requestAnimationFrame.

Signal Types in Implementation

Signal Implementation Value Range
Audio Tone.js audio nodes (Oscillator, Filter, etc.) -1.0 to 1.0 (waveform)
CV Tone.Signal<'number'> Control values (pitch in cents, envelope 0–1, etc.)
Gate Tone.Signal<'number'> 0 = off, 1 = on

06 State Management

Zustand store (src/store/rackStore.ts) using subscribeWithSelector middleware.

interface RackStore {
  // State
  modules: Record<string, RackModule>    // instanceId → module
  connections: Connection[]
  selectedCableId: string | null
  draggingCable: DraggingCable | null

  // Module actions
  addModule(type, position) → instanceId
  removeModule(instanceId) → void       // also removes connected cables
  moveModule(instanceId, position) → void
  setParameter(instanceId, parameterId, value) → void

  // Connection actions
  addConnection(source, dest) → connectionId | null
  removeConnection(connectionId) → void
  selectCable(connectionId | null) → void

  // Cable drag
  startCableDrag(moduleId, portId, signalType) → void
  endCableDrag() → void

  // Patch management
  exportPatch() → string (JSON)
  importPatch(json: string) → void
}

Key Details

  • Instance IDs: {type}-{counter} (e.g., oscillator-1, filter-2)
  • Connection IDs: {srcModule}:{srcPort}→{destModule}:{destPort}
  • addConnection validates: no duplicates, no self-loops, signal type determined from source port.
  • removeModule cascades: removes all connections involving that module.
  • Registry lookup is injected via setRegistryLookup() to avoid circular dependencies between store and engine.

Persistence

Implemented in src/store/persistence.ts:

  • initAutosave() — subscribes to store, debounces 500ms, saves to localStorage['patchglow-autosave']
  • loadAutosave() — restores from localStorage on app start (before engine init)
  • exportPatch() / importPatch() — JSON serialization of { modules, connections }
  • Import restores module counters from instance IDs so new modules get unique IDs

07 Cable System

Cables are rendered as an SVG overlay (src/ui/Cables/) positioned absolutely over the rack. pointerEvents: none on the SVG root, pointerEvents: auto on individual cable <g> elements.

Bezier Curve Calculation

Computed in Cable.tsx: getCablePath():

dx    = end.x - start.x
dy    = end.y - start.y
dist  = sqrt(dx² + dy²)
droop = min(dist * 0.3, 120)     // gravity sag, capped

cp1 = (start.x + dx*0.25,  start.y + droop)
cp2 = (start.x + dx*0.75,  end.y   + droop)
→ M start C cp1, cp2, end

Multi-Layer Rendering

Each cable draws 4 SVG paths for depth and interactivity:

# Layer Stroke Width Purpose
1 Hit area 12 Transparent, easy click targeting
2 Cable body 4–5 Accent color, opacity 0.25–0.5, drop-shadow glow
3 Cable core 2–2.5 Accent color, full opacity, stronger glow
4 Signal flow 2 White, dasharray="4 20", animated via CSS keyframes

Signal Flow Animation

@keyframes cableFlow {
  to { stroke-dashoffset: -24; }
}

Duration varies by signal type: audio = 0.4s (fast), cv = 1.8s (slow), gate = 0.7s (medium).

Color Coding

{ audio: '#ff6b35', cv: '#00e5ff', gate: '#ff2ecb' }

Connection Validation

Implemented in CableLayer.tsx:

  • Drag starts from output port → preview cable follows cursor
  • On drop, elementFromPoint() hit-tests to find destination port element
  • Walks up DOM to find data-module-id, data-port-id, data-direction attributes
  • Validates: direction must be 'input', signal types must be compatible (matching types only)
  • Calls addConnection() on the store

Cable Selection

Click toggles selection via selectCable(). Delete / Backspace removes selected cable. Escape deselects.

08 Visualization System

Per-module Canvas 2D rendering at ~60fps. Each module's visualization component uses the useAnimationFrame hook (src/modules/_shared/useAnimationFrame.ts) which manages a shared requestAnimationFrame loop.

Drawing Utilities

Defined in src/modules/_shared/drawUtils.ts:

Function Description
clearWithFade(ctx, alpha) Semi-transparent dark rect instead of clear — creates a phosphor trail "persistence of vision" effect. Default alpha 0.25.
clearCanvas(ctx) Full clear with dark background fill.
drawGrid(ctx, divisions, color) Faint grid lines with a brighter center zero-reference line.
drawGlowLine(ctx, points, color, thickness) Three-pass neon glow — wide dim outer glow, medium mid glow, thin bright core.
drawWaveform(ctx, data, color) Maps Float32Array samples (-1 to 1) to canvas coordinates. Uses findZeroCrossing() for stable triggered display.
drawSpectrum(ctx, data, color) FFT data as vertical bars with glow effect and shadowBlur on peaks.
findZeroCrossing(data) Finds first negative→positive zero crossing for triggered oscilloscope display.
mapRange(value, ...) Linear interpolation utility for value mapping.

Data Flow

  1. Module engine creates Tone.Analyser nodes (waveform + FFT)
  2. AudioEngineManager.getVisualizationData(instanceId) calls engine.getVisualizationData()
  3. Engine reads analyser data (analyser.getValue()) and returns VisualizationData
  4. Visualization component calls this in its RAF callback and draws to canvas

CRT / Phosphor Aesthetic

  • clearWithFade() creates trailing afterglow on waveform traces
  • drawGlowLine() multi-pass rendering creates the neon phosphor effect
  • CSS class vizScreen on canvas containers provides dark inset background with subtle border
  • Module panels use CSS custom properties from src/theme/variables.css for consistent dark theme

09 How to Add a New Module

Follow these steps to implement a new PatchGlow module. Each module lives in its own folder under src/modules/.

  1. Create Folder

    Create src/modules/<name>/ with four files: definition.ts, engine.ts, Visualization.tsx, index.ts.

  2. definition.ts — Define the Module Interface

    import type { ModuleDefinition } from '@/types/module'

const myModuleDefinition: ModuleDefinition = {
  type: 'my-module',
  name: 'My Module',
  description: 'What this module does',
  learningSummary: 'What synthesis concept this teaches',
  hp: 10,
  category: 'modifier',
  accentColor: '#ff00ff',
  ports: [
    { id: 'in',  label: 'IN',  direction: 'input',  signalType: 'audio' },
    { id: 'out', label: 'OUT', direction: 'output', signalType: 'audio' },
  ],
  parameters: [
    { id: 'amount', label: 'AMT', type: 'knob',
      min: 0, max: 1, default: 0.5, curve: 'linear' },
  ],
}
export default myModuleDefinition

  3. engine.ts — Implement the Tone.js Audio Engine

    import * as Tone from 'tone'
import type { ModuleAudioEngine, VisualizationData } from '@/types/module'

export class MyModuleEngine implements ModuleAudioEngine {
  private node: Tone.ToneAudioNode | null = null
  private analyser: Tone.Analyser | null = null

  initialize(context: Tone.BaseContext): void {
    // Create Tone.js nodes
    this.analyser = new Tone.Analyser('waveform', 256)
  }

  getOutputNode(portId: string) { /* return appropriate node */ }
  getInputNode(portId: string)  { /* return appropriate node */ }

  setParameter(id: string, value: number | string): void {
    // Update Tone.js node parameters
  }

  getVisualizationData(): VisualizationData {
    return {
      waveform: this.analyser?.getValue() as Float32Array,
    }
  }

  dispose(): void {
    // Clean up all Tone.js nodes
    this.node?.dispose()
    this.analyser?.dispose()
  }
}

  4. Visualization.tsx — Canvas Rendering

    import { useRef } from 'react'
import { useAnimationFrame } from '@/modules/_shared/useAnimationFrame'
import { clearWithFade, drawWaveform } from '@/modules/_shared/drawUtils'
import AudioEngineManager from '@/engine/AudioEngineManager'
import panelStyles from '@/ui/ModulePanel/ModulePanel.module.css'

export default function MyModuleVisualization({ moduleId, accentColor }) {
  const canvasRef = useRef<HTMLCanvasElement>(null)

  useAnimationFrame(() => {
    const ctx = canvasRef.current?.getContext('2d')
    if (!ctx) return
    clearWithFade(ctx)
    const { waveform } = AudioEngineManager.getInstance()
      .getVisualizationData(moduleId)
    if (waveform) drawWaveform(ctx, waveform, accentColor)
  })

  return (
    <div className={panelStyles.vizScreen}>
      <canvas ref={canvasRef}
        width={180} height={80} />
    </div>
  )
}

  5. index.ts — Bundle as ModuleRegistration

    import type { ModuleRegistration } from '@/types/module'
import definition from './definition'
import { MyModuleEngine } from './engine'
import Visualization from './Visualization'

const registration: ModuleRegistration = {
  definition,
  createEngine: () => new MyModuleEngine(),
  VisualizationComponent: Visualization,
}
export default registration

  6. Register in App.tsx

    import myModuleReg from '@/modules/my-module'
registerModule(myModuleReg)

Checklist

  • Clear learningSummary — what synthesis concept does this teach?
  • Visualization reveals the module's function in real time
  • All knobs produce visible changes in the visualization
  • Works with at least one example patch
  • TypeScript strict mode — no any types
  • Proper dispose() cleanup of all Tone.js nodes

10 Known Limitations & Future Work

Current Limitations

  • Module count: ~15–20 modules work smoothly on a modern machine. Each adds Canvas rendering and Tone.js nodes.
  • Browser compatibility: Best in Chrome (strongest Web Audio support). Firefox works well. Safari has occasional Web Audio quirks.
  • Mobile / touch: Basic touch support exists (pointer events, touch cable dragging). Not optimized for small screens.
  • No undo/redo: State changes are one-way. Use Save/Load for checkpoints.
